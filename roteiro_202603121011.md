## Fase 2 — Tela de Layout de Importação

> Substituir o formulário inline pela tela dedicada.

### 2.1 — Componente `ImportLayoutFormComponent`

**Arquivo:** `import-layout/import-layout-form.component.ts`

**Seções do formulário:**

**a) Identificação**
- `nome` (required), `descricao`
- `cnpj` readonly da sessão

**b) Configuração da Planilha**
- `linha_cabecalho` (number, default 1)
- `linha_inicio_dados` (number, default 2)
- `nome_aba` (optional text)

**c) Mapeamento de Colunas**
- `FormArray` de `ColunaLayout`
- Select "Campo Destino" populado via `camposDisponiveis()`
- Select "Tipo de Dado" fixo: `string`, `decimal`, `date`, `integer`
- Input "Coluna Excel" (ex: A, B, C2)
- Input "Formato" (visível quando tipo = `date`)
- Botões: `+ Adicionar Coluna`, `× Remover`

**d) Botões**
- `Salvar` → POST ou PUT dependendo se `id` existe
- `Cancelar` → navega para `/extrato/import-layout`

### 2.2 — Atualização da Lista (`ImportLayoutComponent`)

- Botão "Editar" → navega para `/extrato/import-layout/:id/edit`
- Botão "Novo Layout" → navega para `/extrato/import-layout/new`
- Remover formulário inline da tela de listagem

### 2.3 — Componente `ColunaLayoutTableComponent` *(shared reutilizável)*

Componente separado para a tabela de mapeamento de colunas, para ser reutilizado no Wizard (Fase 4).

**Entregáveis da Fase 2:**

- [ ] `coluna-layout-table.component.ts` (shared)

## Fase 4 — Wizard de Upload com Criação Dinâmica de Layout

> Fluxo de upload aprimorado; fluxo de criação de layout a partir do Excel.

### 4.1 — Refatoração do `UploadComponent`

Dividir a tela atual em um fluxo de **etapa única com painel lateral de contexto**:

```
Etapa 1 (sempre visível):
  [ Dropzone ]
  [ Mês ] [ Ano ]
  [ Layout de Importação ▼ ] [ + Criar novo ]
  [ Layout de Saída ▼ ]
  [ ⬆ Processar ]
```

- Quando `layout_id` não selecionado e arquivo carregado: habilitar botão "+ Criar novo"
- Quando nenhum layout disponível: exibir aviso com link direto ao wizard

### 4.2 — `LayoutWizardComponent` (modal em múltiplos passos)

**Arquivo:** `upload/layout-wizard.component.ts`

**Passo 1 — Configuração da Planilha**
- Campos: `nome`, `linha_cabecalho`, `linha_inicio_dados`, `nome_aba`
- Botão "Pré-visualizar" → chama `previewExcel()` → exibe `ExcelPreviewTableComponent`
- Botão "Próximo" (habilitado após preview ou preenchimento manual)

**Passo 2 — Mapeamento de Colunas**
- Reutiliza `ColunaLayoutTableComponent` (Fase 2)
- Preview do Excel sempre visível ao lado/acima do mapeamento
- Botão "Testar Parse" → chama `testParse()` → avança para Passo 3

**Passo 3 — Preview de Lançamentos + Contas Pendentes**
- Tabela de lançamentos parseados
- Se `contas_pendentes.length > 0`: exibir seção de mapeamento manual
  ```
  | CONTA SEM DEFINIÇÃO | DEFINIR CONTA |
  | 2004                | [ input ]     |
  ```
- Botão "Testar novamente" (atualiza com mapeamentos)
- Botão "Salvar Layout"

**Passo 4 — Confirmação**
- Resumo: nome do layout criado, qtd de colunas, qtd de lançamentos detectados
- Botão "Usar este Layout" → fecha wizard, seleciona o novo layout na Etapa 1 do Upload

### 4.3 — `ExcelPreviewTableComponent` *(shared)*

Componente de tabela para exibir as linhas do Excel com formatação, para ser reutilizado no wizard.

**Entregáveis da Fase 4:**
- [ ] `upload.component.ts` refatorado
- [ ] `layout-wizard.component.ts` (wizard multi-passo)
- [ ] `excel-preview-table.component.ts` (shared)

## Fase 5 — Regras de Processamento

> Acessível a partir da tela de Layout de Importação (link/aba na seção 4).

### 5.1 — `RegraListComponent`

**Arquivo:** `import-layout/regras/regra-list.component.ts`

- Lista as regras do layout (ordem, nome, tipo, ativo)
- Botões: Editar, Mover ↑↓, Deletar, Toggle Ativo
- Botão "Nova Regra" → abre `RegraFormComponent`

### 5.2 — `RegraFormComponent`

**Arquivo:** `import-layout/regras/regra-form.component.ts`

**Seções:**

**a) Identificação**
- `nome`, `descricao`, `tipo` (select: filtro / transformacao / enriquecimento / validacao)

**b) Condições** *(FormArray)*
```
SE   campo [ ▼ ]  operador [ contém ▼ ]  valor [ ______ ]
E    campo [ ▼ ]  operador [ contém ▼ ]  valor [ ______ ]
[ + Adicionar Condição ]
```

**c) Ação**
```
ENTÃO  campo [ conta_credito ▼ ]  recebe  [ 789 ]
```
- `tipo_acao` define o comportamento e campos disponíveis

**d) Ações Extras** *(opcional, collapsible)*
- FormArray de ações adicionais

**Entregáveis da Fase 5:**
- [ ] `regra-list.component.ts`
- [ ] `regra-form.component.ts`
- [ ] Integração com `ImportLayoutFormComponent` (aba ou seção "Regras")
