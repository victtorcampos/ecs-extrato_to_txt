## 1. Problemas Identificados

### [F1] `test-parse` no wizard exibe contas de regras como pendentes (Step 4)

**Componente**: `StepPending.jsx` / `ImportWizard.jsx:handleCheckPendentes`
**Endpoint afetado**: `POST /api/v1/import-layouts/test-parse`

O Step 3 → 4 chama `/test-parse` para pré-visualizar pendências **antes** de criar o lote. O endpoint `test-parse` provavelmente usa a mesma lógica do use case (sem o fix E3). Por isso, contas definidas por `regras_conta` (ex: `18`, `22`) ainda aparecem no Step 4 como pendentes.

**Resultado atual:**

```
Step 4 — Pendências
  ┌────────────────────────────────────────┐
  │  ⚠ Conta 1130 (crédito)  [________]  │ ← correto
  │  ⚠ Conta 18   (débito)   [________]  │ ← incorreto: destino de regra
  │  ⚠ Conta 22   (crédito)  [________]  │ ← incorreto: destino de regra
  └────────────────────────────────────────┘
```

**Resultado esperado:**

```
Step 4 — Pendências
  ┌────────────────────────────────────────┐
  │  ⚠ Conta 1130 (crédito)  [________]  │ ← única pendência real
  └────────────────────────────────────────┘
  ℹ Contas gerenciadas por regras: 18, 22
```

**Solução — 2 caminhos:**

**A) Corrigir o endpoint `test-parse` no backend** (recomendado — mesma lógica do E3):
Localizar o use case / handler de `test-parse` e aplicar a mesma exclusão de `contas_de_regras`.

**B) Filtrar no frontend** (alternativa sem tocar no backend):
```jsx
// ImportWizard.jsx — handleCheckPendentes
const contasDeRegras = new Set(
  regrasContas.flatMap(r => [r.conta_debito, r.conta_credito, ...(r.acoes || []).map(a => a.valor)])
    .filter(Boolean)
);
const pendentes = response.data.contas_pendentes?.filter(p => !contasDeRegras.has(p.conta)) || [];
setContasPendentes(pendentes);
```

### [F2] Wizard não aguarda conclusão do lote — navega imediatamente para a página do lote

**Componente**: `ImportWizard.jsx:handleProcess` (linhas ~204-217)

Após criar o lote, o wizard navega para `/lotes/{id}`. O lote ainda está em `aguardando` ou `processando`. O usuário precisa recarregar a página do lote para ver o status final e só então baixar o arquivo.

**Fluxo atual:**
```
[Processar] → POST /lotes → navigate("/lotes/{id}")
                              → página abre com status "aguardando"
                              → usuário aguarda e recarrega manualmente
```

**Fluxo proposto — Step 7 (Conclusão) no próprio wizard:**
```
[Processar] → POST /lotes → Step 7 (Aguardando...)
                              → polling GET /lotes/{id} a cada 2s
                              → status "concluido" → botão [Baixar TXT]
                              → status "pendente"  → volta para Step 4 com pendências reais
                              → status "erro"      → exibe mensagem de erro
```

**Arquivo**: `ImportWizard.jsx`

**Novos estados:**
```js
const [loteId, setLoteId] = useState(null);
const [loteStatus, setLoteStatus] = useState(null);
const [loteErro, setLoteErro] = useState(null);
```

**Novo handler `handleProcess` (trecho):**
```js
const handleProcess = async () => {
  // ... criar layout e lote (igual ao atual) ...
  const loteRes = await lotesApi.criar({ ... });
  setLoteId(loteRes.id);
  setLoteStatus(loteRes.status);
  setStep(7);
  // Polling
  await pollLote(loteRes.id);
};

const pollLote = async (id) => {
  const MAX = 30;
  for (let i = 0; i < MAX; i++) {
    await new Promise(r => setTimeout(r, 2000));
    const lote = await lotesApi.obter(id);
    setLoteStatus(lote.status);
    if (lote.status === 'concluido') return;
    if (lote.status === 'pendente') {
      // Voltar para step 4 com pendências do lote real
      setContasPendentes(lote.pendencias.filter(p => !p.resolvida));
      setStep(4);
      return;
    }
    if (lote.status === 'erro') {
      setLoteErro(lote.mensagem_erro);
      return;
    }
  }
};
```

**Novo Step 7 (StepConclusao) — JSX em ImportWizard:**
```jsx
{step === 7 && (
  <StepConclusao
    loteId={loteId}
    status={loteStatus}
    erro={loteErro}
    onReiniciar={() => { setStep(1); setLoteId(null); }}
  />
)}
```

**Novo componente `StepConclusao.jsx`:**
```jsx
export const StepConclusao = ({ loteId, status, erro, onReiniciar }) => {
  const downloadUrl = `${API_BASE}/api/v1/lotes/${loteId}/download`;

      <CheckCircle className="w-12 h-12 text-emerald-500 mx-auto" />
      <h2 className="text-lg font-semibold text-slate-900">Lote processado!</h2>

        </a>
        <Button variant="outline" onClick={onReiniciar}>Nova Importação</Button>
      </div>
    </div>
  );

  if (status === 'erro') return (
    <div className="text-center space-y-4">
      <XCircle className="w-12 h-12 text-red-500 mx-auto" />
      <h2 className="text-lg font-semibold">Erro no processamento</h2>
      <p className="text-sm text-slate-600">{erro}</p>
      <Button variant="outline" onClick={onReiniciar}>Recomeçar</Button>
    </div>
  );

  // aguardando / processando
  return (
    <div className="text-center space-y-4">
      <Loader2 className="w-10 h-10 text-slate-400 animate-spin mx-auto" />
      <p className="text-sm text-slate-600">Processando lote...</p>
    </div>
  );
};
```
