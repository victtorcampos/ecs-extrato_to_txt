### FASE 1 — `dominio_sistemas_txt.py` (Bugs E1 e E2)

- [ ] **1.1** Adicionar import de `date` e `datetime` no topo do arquivo
  ```python
  from datetime import date as date_type, datetime as datetime_type
  ```
  > Verificar se já há algum import de `datetime` no arquivo antes de adicionar.

- [ ] **1.2** Substituir o método `_formatar_data` inteiro
  ```python
  def _formatar_data(self, data) -> str:
      """Formata data para DD/MM/AAAA — aceita date, datetime ou string"""
      if not data:
          return ""
      if isinstance(data, (date_type, datetime_type)):
          return data.strftime("%d/%m/%Y")
      data_str = str(data).strip()
      if "/" in data_str and len(data_str) >= 10:
          return data_str[:10]
      if "-" in data_str and len(data_str) >= 10:
          partes = data_str[:10].split("-")
          if len(partes) == 3:
              return f"{partes[2]}/{partes[1]}/{partes[0]}"
      return data_str
  ```

- [ ] **1.3** Mover o registro `6000` para FORA do loop `for lanc in lancamentos:`

  **Antes (com bug):**
  ```python
  for lanc in lancamentos:
      ...
      reg_6000 = self._montar_registro(sep, incluir_delim, [
          "6000", tipo_lanc, "", "", "",
      ])
      linhas.append(reg_6000)

      reg_6100 = self._montar_registro(sep, incluir_delim, [...])
      linhas.append(reg_6100)
  ```

  **Depois (corrigido):**
  ```python
  # Registro 6000 — único, representa o lote inteiro
  reg_6000 = self._montar_registro(sep, incluir_delim, [
      "6000", tipo_lanc, "", "", "",
  ])
  linhas.append(reg_6000)

  for lanc in lancamentos:
      conta_deb = mapeamentos.get(lanc.conta_debito, lanc.conta_debito)
      conta_cred = mapeamentos.get(lanc.conta_credito, lanc.conta_credito)
      valor_formatado = self._formatar_valor(lanc.valor)
      data_formatada = self._formatar_data(lanc.data)
      historico = lanc.historico or ""

      reg_6100 = self._montar_registro(sep, incluir_delim, [
          "6100",
          data_formatada,
          conta_deb,
          conta_cred,
          valor_formatado,
          cod_hist_padrao,
          historico,
          nome_usuario,
          cod_filial,
          "",
      ])
      linhas.append(reg_6100)
  ```

## Considerações Finais

1. **FASE 1 não tem efeito colateral** — `_formatar_data` mais permissivo (aceita `date`, `datetime` e `str`) não quebra nenhum outro uso; mover `6000` para fora do loop só afeta o arquivo gerado.

2. **FASE 2 busca o layout uma segunda vez** no método `executar` (já foi buscado em `_parse_excel`). Para evitar a busca duplicada, pode-se refatorar `_parse_excel` para retornar `(lancamentos, layout)` — porém isso é opcional e aumenta o escopo. A dupla busca é segura e o impacto de performance é desprezível.

3. **`contas_de_regras` vazio não quebra nada** — se `layout_repository` for `None` ou o layout não for encontrado, o set fica vazio e o comportamento é idêntico ao atual.

4. **Sem mudança de interface pública** — nenhum DTO, endpoint ou contrato externo é alterado. A correção é totalmente interna ao backend.
