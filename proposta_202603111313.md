## 2. Bugs Encontrados

### [E1] BUG CRÍTICO — `_formatar_data` não suporta objeto `date` Python

**Arquivo**: `backend/src/adapters/outbound/output_generators/dominio_sistemas_txt.py:118`
**Gravidade**: Crítica — o gerador falha completamente, cai no fallback

**Causa raiz:**
`DynamicExcelParser` retorna `Lancamento.data` como `datetime.date` (objeto Python).
O método `_formatar_data` recebe esse objeto e tenta fazer `"/" in data_str`, o que levanta `TypeError: argument of type 'datetime.date' is not iterable`.

Esse erro é capturado em `_gerar_saida`:

```python
# usecases.py:241
except Exception as e:
    logger.warning(f"Erro ao usar perfil de saída ..., usando TXT padrão: {e}")
# → cai no txt_generator.gerar() padrão → formato errado
```

**Código atual (com bug):**
```python
def _formatar_data(self, data_str: str) -> str:
    if not data_str:
        return ""
    if "/" in data_str and len(data_str) >= 10:   # ← TypeError se data_str é date!
        return data_str[:10]
    if "-" in data_str and len(data_str) >= 10:
        partes = data_str[:10].split("-")
        if len(partes) == 3:
            return f"{partes[2]}/{partes[1]}/{partes[0]}"
    return data_str
```

**Correção:**
```python
def _formatar_data(self, data) -> str:
    from datetime import date as date_type, datetime as datetime_type
    if not data:
        return ""
    if isinstance(data, (date_type, datetime_type)):
        return data.strftime("%d/%m/%Y")
    data_str = str(data).strip()
    if "/" in data_str and len(data_str) >= 10:
        return data_str[:10]
    if "-" in data_str and len(data_str) >= 10:
        partes = data_str[:10].split("-")
        if len(partes) == 3:
            return f"{partes[2]}/{partes[1]}/{partes[0]}"
    return data_str
```

### [E2] BUG CRÍTICO — Um registro `6000` por lançamento em vez de um por lote

**Arquivo**: `backend/src/adapters/outbound/output_generators/dominio_sistemas_txt.py:47`
**Gravidade**: Crítica — estrutura do arquivo diverge do formato Domínio Sistemas

**Causa raiz:**
O loop `for lanc in lancamentos:` emite `6000` + `6100` para **cada** lançamento.
O formato correto do Domínio Sistemas é: **1 registro `6000`** (cabeçalho do lote) seguido de **N registros `6100`** (um por lançamento).

**Saída atual (com bug):**
```text
|0000|01293422000165|
|6000|X||||            ← lançamento 1
|6100|01/01/2026|12|78|60000,00|...|
|6000|X||||            ← lançamento 2 — ERRADO
|6100|01/01/2026|78|25|14073,93|...|
|6000|X||||            ← lançamento 3 — ERRADO
|6100|01/01/2026|18|22|1438,71|...|
```

**Saída esperada:**
```text
|0000|01293422000165|
|6000|X||||            ← único, para o lote inteiro
|6100|01/01/2026|12|78|60000,00|...|
|6100|01/01/2026|78|25|14073,93|...|
|6100|01/01/2026|18|22|1438,71|...|
```

**Código atual (com bug):**
```python
for lanc in lancamentos:
    ...
    reg_6000 = self._montar_registro(sep, incluir_delim, ["6000", tipo_lanc, "", "", ""])
    linhas.append(reg_6000)   # ← dentro do loop → um por lançamento

    reg_6100 = self._montar_registro(sep, incluir_delim, [...])
    linhas.append(reg_6100)
```

**Correção:**
```python
# Registro 6000 — uma única vez, antes do loop
reg_6000 = self._montar_registro(sep, incluir_delim, ["6000", tipo_lanc, "", "", ""])
linhas.append(reg_6000)

for lanc in lancamentos:
    conta_deb = mapeamentos.get(lanc.conta_debito, lanc.conta_debito)
    conta_cred = mapeamentos.get(lanc.conta_credito, lanc.conta_credito)
    valor_formatado = self._formatar_valor(lanc.valor)
    data_formatada = self._formatar_data(lanc.data)
    historico = lanc.historico or ""

    reg_6100 = self._montar_registro(sep, incluir_delim, [
        "6100",
        data_formatada,
        conta_deb,
        conta_cred,
        valor_formatado,
        cod_hist_padrao,
        historico,
        nome_usuario,
        cod_filial,
        "",
    ])
    linhas.append(reg_6100)
```

### [E3] BUG COMPORTAMENTAL — Contas definidas por `regras_conta` marcadas como pendentes

**Arquivo**: `backend/src/application/usecases/usecases.py:133`
**Gravidade**: Média — gera fricção desnecessária, mas o resultado final pode ser correto após resolução manual

**Causa raiz:**
O `ProcessarLoteUseCase` verifica se **todas** as contas de **todos** os lançamentos estão no `mapeamentos_dict`. Isso inclui contas que foram substituídas pela `regras_conta` do `DynamicExcelParser` (ex: `18` e `22` da regra RECEBTO). Essas contas já são os destinos finais e não precisam de mapeamento — o gerador aplica `mapeamentos.get(conta, conta)` (passthrough se não encontrado).

**Resultado atual:**
- Conta `1130` → pendente ✓ (correto — precisa ser mapeada para `25`)
- Conta `18` → pendente ✗ (incorreto — já é o destino da regra)
- Conta `22` → pendente ✗ (incorreto — já é o destino da regra)

**Impacto:** Usuário precisa criar mapeamentos `18→18` e `22→22` que são semanticamente vazios.

**Código atual (com bug):**
```python
# usecases.py:133-154
for lanc in lancamentos:
    if lanc.conta_debito and lanc.conta_debito not in mapeamentos_dict:
        contas_sem_mapeamento.add((lanc.conta_debito, "debito"))
    if lanc.conta_credito and lanc.conta_credito not in mapeamentos_dict:
        contas_sem_mapeamento.add((lanc.conta_credito, "credito"))
```

**Correção — opção A (mais simples):** tratar mapeamento como passthrough — se a conta não está no dict, não é pendência; o gerador já faz `get(conta, conta)`. Remover completamente a checagem de pendências:

```python
# Sem pendências: o gerador usa passthrough para contas não mapeadas
contas_sem_mapeamento = set()
# (não criar pendências — apenas gerar o arquivo)
```

> ⚠️ Essa opção elimina o conceito de pendências do fluxo. Adequada se o sistema destino (Domínio) aceita contas passadas diretamente.

**Correção — opção B (preserva pendências, exclui contas de regras):**
Coletar as contas que foram setadas por `regras_conta` e excluí-las da checagem:

```python
# Coletar contas setadas por regras (são destinos, não precisam de mapeamento)
contas_de_regras = set()
if layout and layout.regras_conta:
    for regra in layout.regras_conta:
        if regra.conta_debito:
            contas_de_regras.add(regra.conta_debito)
        if regra.conta_credito:
            contas_de_regras.add(regra.conta_credito)
        for acao in regra.acoes:
            if acao.campo_destino in ("conta_debito", "conta_credito") and acao.valor:
                contas_de_regras.add(acao.valor)

# Checar apenas contas que não são destinos de regras
for lanc in lancamentos:
    if lanc.conta_debito and lanc.conta_debito not in mapeamentos_dict \
            and lanc.conta_debito not in contas_de_regras:
        contas_sem_mapeamento.add((lanc.conta_debito, "debito"))
    if lanc.conta_credito and lanc.conta_credito not in mapeamentos_dict \
            and lanc.conta_credito not in contas_de_regras:
        contas_sem_mapeamento.add((lanc.conta_credito, "credito"))
```

> ✅ Opção B recomendada: preserva a lógica de pendências para contas genuinamente não mapeadas.
