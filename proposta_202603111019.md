## 1. Fluxo Atual - Problema

### **Cenário: Reprocessar arquivo com CNPJ 01.293.422/0001-65**

**Layout salvo no sistema**:
```
Nome: "Layout Itaú 2026"
├─ Colunas Mapeadas (col 4→data, col 6→débito, col 7→crédito, col 11→valor)
├─ Regras de Contas (Se histórico contém "RECEBTO" → conta 18/22)
└─ Mapeamentos de Contas (2004→78, 3145→12) ← SEPARADOS
```

**Problem: No Step 2 "Revisão Layout Detectado"**
1. ✅ Layout aparece na lista para reusar
2. ✅ Ao clicar "Aplicar" → colunas preenchidas automaticamente
3. ❌ Mapeamentos de contas NÃO são carregados
4. ❌ Usuário precisa digitar 2004→78 e 3145→12 novamente no Step 4

**Impacto**: Tempo de reprocessamento ~5 min (deveria ser ~1-2 min)

```
Step 1 (Upload) → POST /detect → Step 2 (Review) → Step 3 (Accounts) → Step 4 (Pending) → Step 5 (Preview) → Step 6 (Download+Process)
```

| Step | Componente | O que faz hoje | Problema |
|---|---|---|---|
| 1 | `StepUpload` | Recebe CNPJ, período, arquivo | — |
| 2 | `StepReview` | Exibe colunas detectadas automaticamente | **Não oferece layouts salvos do CNPJ** |
| 3 | `StepAccounts` | Editor de `regrasContas` com templates sugeridos | **Inicia vazio — não carrega `regras_conta` do layout salvo** |
| 4 | `StepPending` | Resolve contas sem mapeamento | — |
| 5 | `StepPreview` | Preview dos lançamentos | — |
| 6 | `StepDownload` | Escolhe perfil de saída e processa | **Sempre cria layout `Auto-{filename}-timestamp`, mesmo se reutilizou existente** |

### Página LayoutDetail (`/layouts/:id`)

- Exibe dados do layout + regras de transformação/filtro (via `regrasApi` → `/rules`)
- **`AccountRulesBuilder` existe mas NÃO é usado aqui** — só em `LayoutForm`
- Logo: `regras_conta` salvas no layout não são editáveis nessa página

  const mapaLayout = {};

## 6. Decisões de Implementação

1. **Merge de colunas por `coluna_excel`** (índice): é a chave natural para correlacionar colunas detectadas com colunas do layout salvo. Funciona bem quando o arquivo tem a mesma estrutura.

2. **`regrasContas` pré-carregadas mas editáveis**: o usuário pode modificar as regras mesmo após aplicar o layout — o `StepAccounts` não muda.

3. **Atualizar vs. criar**: se `layoutAplicadoId` está definido, atualizar o layout existente é o comportamento esperado (evita proliferação de layouts `Auto-*`). O nome do layout é preservado.

4. **`AccountRulesBuilder` em `LayoutDetail`**: usa estado local `regrasContaEdit` inicializado com `layout.regras_conta`. Save separado para não misturar com save das colunas.

### Reprocessamento: Antes vs. Depois

#### **ANTES (Problema)**
```
1. Upload arquivo → Step 2
2. Step 2: Auto-detect colunas
3. Clicar "Aplicar Layout Itaú"
   ✓ Colunas preenchidas
   ✓ Regras carregadas (Step 3 será pré-preenchido)
   ✗ Mapeamentos NÃO carregados
4. Step 3: Editar regras (mantém `regrasContas` do layout)
5. Step 4: Rodar test-parse
   → Contas 2004 e 3145 aparecem como PENDENTES
   → Usuário digita 2004→78 e 3145→12 manualmente
   → 2.5 min neste step sozinho ❌
6. Step 5-6: Normal
```

#### **DEPOIS (Solução)**
```
1. Upload arquivo → Step 2
2. Step 2: Auto-detect colunas + carrega layouts do CNPJ
3. Clicar "Aplicar Layout Itaú"
   ✓ Colunas preenchidas (coluna_excel → campo_destino)
   ✓ Regras carregadas (Step 3 será pré-preenchido)
   ★ NOVO: Carrega `regras_conta` com mapeamentos de contas
4. Step 3: Editar regras (opcional — regras já estão)
5. Step 4: Rodar test-parse
   → Contas 2004 e 3145 JÁ estão pré-mapeadas (2004→78, 3145→12)
   → Se houver outras contas sem mapeamento → usuário resolve só elas
   → 0.5 min neste step ⚡
6. Step 5-6: Normal
```

**Ganho**: ⚡⚡ 80% mais rápido (4 min → 0.5 min no Step 4)

## 10. Conclusões

✅ **Problema bem diagnosticado**: Layouts reutilizados não carregam `regras_conta` no wizard.

✅ **Backend já suporta**: `layout.regras_conta` já é persistido e retornado pela API.

✅ **Frontend pode implementar em 1 dia**: Merge de colunas + carregar `regrasContas` no state.

✅ **Impacto alto**: Reduz tempo de reprocessamento em 80%+ (erro zero, sem digitação repetida).

⚠️ **Considerações**:
- Merge funciona bem se arquivo mantém mesma estrutura (coluna_excel)
- Se arquivo mudar drasticamente, usuário pode ajustar manualmente
- Se aplicou layout e depois removeu, `layoutAplicadoId` permanece (pode criar novo layout)

🎯 **Recomendação**: Implementar conforme proposto. Arquitetura é simples, mudança é only frontend, e benefício é muito grande.
