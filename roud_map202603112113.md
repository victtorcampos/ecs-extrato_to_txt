# Roadmap FrontendV2 — Angular 21
> Data: 2026-03-12 | Sprint Base: 1 semana por fase

---

## Fase 0 — Fundação (Dia 1–2)

### 0.1 Design System & Tokens
- [ ] Configurar variáveis CSS globais (cores, tipografia, espaçamento) em `styles.css`
- [ ] Importar fontes: Manrope, IBM Plex Sans, JetBrains Mono via Google Fonts
- [ ] Configurar Tailwind theme extension com os tokens do `design_guidelines.json`

### 0.2 Core Module
- [ ] `SessionService` — signal de sessão ativa, persistência em `localStorage`
- [ ] `ApiService` — wrapper HttpClient com base URL configurável
- [ ] `CnpjInterceptor` — injeta header `X-CNPJ` automaticamente
- [ ] `APP_INITIALIZER` — carrega sessão do storage na inicialização

### 0.3 Shared Components Base
- [ ] `SpinnerComponent` — loading inline e overlay
- [ ] `BadgeComponent` — status: success / warning / error / info
- [ ] `StatCardComponent` — card de estatística (ícone + label + valor)
- [ ] `ConfirmDialogComponent` — CDK Dialog para confirmações destrutivas

---

## Fase 1 — Shell & Navegação (Dia 2–3)

### 1.1 Layout Shell
- [ ] `ShellComponent` — wrapper principal com sidebar + topbar + outlet
- [ ] `SidebarComponent` — menu lateral com 2 itens: Dashboard / Extrato to TXT
  - [ ] Estado colapsado/expandido com signal
  - [ ] Active route highlight
- [ ] `TopbarComponent`
  - [ ] Logo / breadcrumb dinâmico
  - [ ] Seletor de CNPJ (dropdown com lista de `GET /api/v1/import-layouts/cnpjs`)
  - [ ] Indicador de sessão ativa (chip com CNPJ formatado)

### 1.2 Roteamento Base
- [ ] Configurar `app.routes.ts` com lazy loading
- [ ] `SessionGuard` bloqueando `/extrato/**` sem CNPJ ativo
- [ ] Rota de fallback (404 → dashboard)

## Fase 3 — Feature Extrato to TXT (Dia 4–7)

### 3.1 Extrato Shell
- [ ] `ExtratoShellComponent`
  - [ ] Linha de stat cards (Total Lotes / Concluídos / Pendentes / Processando)
  - [ ] Subnav: `Upload` | `Lotes` com active underline
  - [ ] Botão ⚙ Engrenagem com dropdown: Mapeamento / Layout de Importação / Layout de Saída
  - [ ] `<router-outlet>` para sub-rotas

### 3.2 Upload
- [ ] `UploadComponent`
  - [ ] Drop zone com borda dashed + ícone + instruções
  - [ ] Seleção de arquivo XLS/XLSX
  - [ ] Preview do arquivo selecionado
  - [ ] Seleção de Import Layout (GET /api/v1/import-layouts)
  - [ ] Seleção de Output Profile (GET /api/v1/output-profiles)
  - [ ] Submit: `POST /api/v1/lotes`
  - [ ] Toast de sucesso com link para o lote criado

### 3.3 Lotes
- [ ] `LotesComponent` — listagem
  - [ ] Tabela com colunas: Protocolo / Data / Status / CNPJ / Ações
  - [ ] Filtros: status, data
  - [ ] Paginação
  - [ ] Badge colorido por status
  - [ ] Ação: Download TXT / Reprocessar / Deletar
- [ ] `LoteDetailComponent` — detalhe
  - [ ] Resumo do lote
  - [ ] Lista de pendências (se houver)
  - [ ] `POST /api/v1/lotes/{id}/resolver-pendencias`

### 3.4 Configurações (via botão ⚙)

#### 3.4.1 Mapeamento
- [ ] `MapeamentoComponent`
  - [ ] Listagem de mapeamentos do CNPJ ativo
  - [ ] CRUD: criar / editar / deletar
  - [ ] Bulk update inline

#### 3.4.2 Layout de Importação
- [ ] `ImportLayoutComponent`
  - [ ] Listagem de layouts
  - [ ] Formulário de criação/edição
    - [ ] Preview Excel (`POST /api/v1/import-layouts/preview-excel`)
    - [ ] Mapeamento de colunas
    - [ ] Regras de processamento (CRUD de rules)
  - [ ] Clone de layout (`POST /api/v1/import-layouts/{id}/clone`)

#### 3.4.3 Layout de Saída
- [ ] `OutputLayoutComponent`
  - [ ] Listagem de perfis de saída
  - [ ] Formulário de criação/edição
  - [ ] Sistemas disponíveis (`GET /api/v1/output-profiles/sistemas-disponiveis`)

## Fase 4 — Qualidade & Polish (Dia 8–9)

### 4.1 Acessibilidade
- [ ] Revisão WCAG AA: contraste, foco, ARIA labels
- [ ] Navegação por teclado em dropdowns e dialogs

### 4.2 Performance
- [ ] OnPush em todos os componentes (auditoria)
- [ ] TrackBy em todos os `@for`
- [ ] Lazy loading verificado no bundle analyzer

### 4.3 Responsividade
- [ ] Sidebar colapsável em mobile
- [ ] Stat cards em grid responsivo (1→2→4 colunas)
- [ ] Tabelas com scroll horizontal em telas pequenas

### 4.4 Estados Vazios & Erros
- [ ] Empty state para lista de lotes vazia
- [ ] Error boundary com mensagem amigável
- [ ] Skeleton loading nos cards e tabelas
