# Proposta de Desenvolvimento — Frontend v2
**Ref:** `proposta_202603121011`
**Data:** 2026-03-12

---

## Contexto

O backend já possui endpoints completos e maduros para:
- `POST /api/v1/import-layouts/preview-excel` — pré-visualização da planilha Excel (abas, cabeçalhos, linhas de dados)
- `POST /api/v1/import-layouts/detect` — detecção automática de estrutura a partir do Excel
- `POST /api/v1/import-layouts/test-parse` — preview completo dos lançamentos parseados, com contas pendentes
- `GET/POST/PUT/DELETE /api/v1/import-layouts/{id}/rules` — CRUD completo de regras de processamento

O frontend ainda não consome esses endpoints. Esta proposta cobre a exposição dessas capacidades ao usuário final.

---

## Escopo da Proposta

### 1. Formulário de Layout de Importação — Tela Dedicada

**Situação atual:** formulário inline com apenas `nome`, `cnpj`, `descricao`.

**Proposta:** substituir o formulário inline por uma **tela/rota dedicada** (`/extrato/import-layout/:id/edit` e `/extrato/import-layout/new`) com as seções:

#### Seção 1 — Identificação
| Campo | Tipo | Obrigatório |
|---|---|---|
| Nome | text | sim |
| CNPJ | readonly (sessão) | — |
| Descrição | text | não |

#### Seção 2 — Configuração da Planilha
| Campo | Tipo | Obrigatório |
|---|---|---|
| Linha do Cabeçalho | number | sim |
| Linha Inicial dos Dados | number | sim |
| Nome da Aba (opcional) | text | não |

#### Seção 3 — Mapeamento de Colunas
Tabela editável inline:

| # | Coluna Excel | Campo Destino | Tipo de Dado | Formato | Obrigatório |
|---|---|---|---|---|---|
| A | DATA PGTO | data | date | DD/MM/YYYY | sim |
| B | CONTA C | conta_credito | string | — | sim |
| C | CONTA D | conta_debito | string | — | sim |
| D | VALOR | valor | decimal | — | sim |
| E | DESCRIÇÃO | historico | string | — | não |

- **Campo Destino**: select com valores vindos de `GET /api/v1/import-layouts/campos-disponiveis`
- **Tipo de Dado**: select fixo (`string`, `decimal`, `date`, `integer`)
- Botões: `+ Adicionar Coluna` / `× Remover`

#### Seção 4 — Regras de Processamento *(somente em edição)*
Lista das regras vinculadas ao layout (`GET /rules`), com botões para criar, editar, reordenar e deletar.

### 2. Formulário de Layout de Saída — Tela Dedicada

**Situação atual:** formulário inline com `nome`, `sistema_destino`, `formato`, `descricao`.

**Proposta:** tela dedicada (`/extrato/output-layout/:id/edit`) com as seções:

#### Seção 1 — Identificação
| Campo | Tipo |
|---|---|
| Nome | text |
| Descrição | text |

#### Seção 2 — Configuração de Destino
| Campo | Tipo |
|---|---|
| Sistema Destino | select (via API `sistemas-disponiveis`) |
| Formato | select dinâmico (formatos do sistema selecionado) |
| Ativo | toggle |
| Padrão | toggle |

#### Seção 3 — Configurações Avançadas *(opcional, collapsible)*
Campos `config` específicos de cada sistema, renderizados dinamicamente conforme os metadados do backend.

### 3. Wizard de Upload com Criação Dinâmica de Layout

**Situação atual:** upload envia o arquivo diretamente; se não houver layout, o processamento falha silenciosamente.

**Proposta:** novo fluxo de upload em **3 etapas**:

#### Etapa 1 — Arquivo + Período + Seleção de Layouts
```
[ Dropzone: arrastar/clicar — .xls, .xlsx ]

[ MÊS ]    [ ANO ]

[ LAYOUT DE IMPORTAÇÃO  ▼ ]   [ + Criar novo layout ]
[ LAYOUT DE SAÍDA       ▼ ]

[ ⬆ Processar Extrato ]
```

Quando o usuário clicar em **"+ Criar novo layout"** (ou não tiver nenhum disponível), iniciar o **Wizard de Criação de Layout**.

#### Etapa 2 — Wizard de Criação de Layout (modal ou rota separada)

**Passo 2a — Configuração da Planilha**
```
[ NOME ]                  [ CNPJ (sessão) ]

[ LINHA CABEÇALHO ]   [ LINHA INICIAL DOS DADOS ]

[ Pré-visualizar Excel ]
```
Ao clicar em "Pré-visualizar", chama `POST /api/v1/import-layouts/preview-excel` com o arquivo já carregado, retornando a grid de dados reais do Excel.

**Passo 2b — Mapeamento de Colunas (com Preview do Excel)**
```
DADOS EXCEL (preview)
| A          | B            | C             | D        | E                                  |
| data       | conta_debito | conta_credito | valor    | historico                          |
| 01/01/2026 | 3145         | 2004          | 60000    | VLR ALUGUEL JAN 2026 ...           |

MAPEAMENTO
| Col Excel | Campo Destino     | Tipo    | Formato      |
|-----------|-------------------|---------|--------------|
| A2        | data              | date    | DD/MM/YYYY   |
| B2        | conta_debito      | string  |              |
| C2        | conta_credito     | string  |              |
| D2        | valor             | decimal |              |
| E2        | historico         | string  |              |
```

**Passo 2c — Preview de Lançamentos**
Chama `POST /api/v1/import-layouts/test-parse` e exibe:
- Tabela de lançamentos parseados
- Contas sem mapeamento (pendentes)

**Passo 2d — Mapeamento de Contas Pendentes**
```
| CONTA SEM DEFINIÇÃO | DEFINIR CONTA |
| 2004                | [ 25     ]    |
| 3145                | [ 45     ]    |
| 1130                | [ 688    ]    |
```

**Passo 2e — Salvar Layout**
`POST /api/v1/import-layouts` → retorna o novo layout → seleciona automaticamente na Etapa 1.

#### Etapa 3 — Processamento
Retorna ao fluxo normal de upload com o layout recém-criado já selecionado.

### 4. Regras de Processamento

Acessível a partir do formulário de Layout de Importação (Seção 4).

**Interface de Regra:**
```
REGRA: [ REGRA001 ]

CONDIÇÕES (todas devem ser verdadeiras):
  SE   campo [ historico     ▼ ]  operador [ contém ▼ ]  valor [ PAGAMENTO      ]
  E    campo [ historico     ▼ ]  operador [ contém ▼ ]  valor [ ABC SERVICOS   ]

ENTÃO:
  campo [ conta_credito ▼ ] recebe valor [ 789 ]

[ + Adicionar Condição ]   [ + Adicionar Ação ]
[ Salvar ]  [ Cancelar ]
```

Operadores suportados pelo backend: `igual`, `diferente`, `maior`, `menor`, `entre`, `contém`, `regex`, etc.

Tipos de ação: `transformacao`, `filtro`, `enriquecimento`, `validacao`.
